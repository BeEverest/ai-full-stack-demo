## 一、产品概述

### 1.1 背景

公司旗下拥有三款面向消费者与企业客户的AI硬件终端产品：AI眼镜、AI机器人、AI玩具。当前缺乏统一的品牌数字入口，需要建立一个企业官网作为品牌展示、产品推广和商业转化的核心阵地。

### 1.2 产品定位

> 一个聚焦「未来科技感」的简洁企业官网，服务于品牌认知建立与产品展示，V1以内容展示为核心，不含电商购买功能。

### 1.3 核心目标

| 目标 | 说明 |
|------|------|
| 品牌认知 | 建立「AI智能终端」的品牌心智，传递科技领先感 |
| 产品展示 | 清晰展示三款产品的核心能力与差异化 |
| 线索获取 | 为B端合作与C端预购留存意向用户信息 |
| SEO基础 | 建立基础的搜索引擎收录能力 |

---

## 二、目标用户

### 2.1 用户画像

**主要用户 — B端决策者**
- 角色：科技公司采购负责人、企业数字化转型负责人
- 诉求：了解产品技术规格、合作方式、品牌背书
- 行为：浏览产品参数、寻找联系方式、下载资料

**次要用户 — C端早期采用者**
- 角色：科技爱好者、25-40岁，关注AI前沿产品
- 诉求：了解产品特性、注册预购意向
- 行为：视频/图片浏览、分享社媒、留下邮箱

**次要用户 — 媒体/投资人**
- 角色：科技媒体记者、早期投资人
- 诉求：公司背景、产品亮点、联系方式
- 行为：快速扫描首页、查阅关于我们

## 四、页面详细需求

### 4.1 首页

**目标：** 10秒内传递「我们做AI硬件终端，很酷」

**Section 布局：**

| # | Section | 内容 | 设计要点 |
|---|---------|------|----------|
| 1 | Hero | 全屏视觉 + Slogan + CTA | 粒子动效/3D场景，强视觉冲击 |
| 2 | 产品矩阵 | 三款产品卡片横排展示 | Hover动效，点击进入产品页 |
| 3 | 核心技术亮点 | 3-4个技术关键词 + 图标 | 数字滚动动效，简洁有力 |
| 4 | 品牌价值主张 | 1-2句话 + 视觉 | 大字排版，强调品牌调性 |
| 5 | 新闻/媒体背书 | 媒体logo或新闻标题 | 灰度logo，信任背书 |
| 6 | CTA召集 | 合作咨询 / 预约体验 | 双按钮，区分B/C端入口 |
| 7 | Footer | 导航、社媒、版权 | 深色背景，简洁布局 |

**Hero Slogan（建议）：**
> "智能，从此触手可及"  
> 副标题：AI眼镜 · AI机器人 · AI玩具

**CTA按钮：**
- 主按钮：「探索产品」→ 跳转产品列表
- 次按钮：「商务合作」→ 跳转联系页

## 五、设计系统

### 5.1 视觉风格

**核心关键词：** 未来感 · 极简 · 科技 · 精密

### 5.2 色彩方案

| 色彩角色 | 色值 | 使用场景 |
|----------|------|----------|
| 主背景色 | `#0A0A0F` 深空黑 | 页面主背景 |
| 次级背景 | `#111827` 深蓝黑 | 卡片、Section区隔 |
| 主强调色 | `#00F5FF` 青蓝 | CTA按钮、高亮、边框发光 |
| 次强调色 | `#7C3AED` 科技紫 | 渐变、装饰元素 |
| 主文字色 | `#F9FAFB` 近白 | 标题、主要文字 |
| 次文字色 | `#9CA3AF` 浅灰 | 描述文字、说明 |
| 边框/分割线 | `#1F2937` | 卡片边框 |

**渐变方案：**
- Hero渐变：`linear-gradient(135deg, #00F5FF 0%, #7C3AED 100%)`
- 发光效果：`box-shadow: 0 0 20px rgba(0, 245, 255, 0.3)`

### 5.3 字体系统

| 层级 | 字体 | 尺寸 | 字重 |
|------|------|------|------|
| 中文主标题 | 思源黑体 / 苹方 | 48-64px | 700 |
| 英文标题 | Inter / Space Grotesk | 48-64px | 700 |
| 副标题 | 同上 | 24-32px | 600 |
| 正文 | 思源宋体 / 系统字体 | 16px | 400 |
| 标注 | 同上 | 14px | 400 |

### 5.4 动效规范

| 场景 | 动效类型 | 参数 |
|------|----------|------|
| 页面进入 | 元素淡入上移 | duration: 600ms, ease-out |
| 滚动触发 | Intersection Observer淡入 | threshold: 0.1 |
| 卡片Hover | 上移 + 边框发光 | translateY(-4px), 200ms |
| CTA按钮 | 发光呼吸效果 | 脉冲动画, 2s循环 |
| Hero背景 | 粒子浮动 或 网格流动 | Canvas/CSS动画 |
| 数字计数 | 滚动到视口时从0计数 | 1.5s ease-out |

### 5.5 组件规范

**按钮：**
- 主按钮：青蓝色背景 + 深色文字 + 发光hover
- 次按钮：透明背景 + 青蓝色边框 + 白色文字
- 圆角：`border-radius: 4px`（硬朗科技感，不用大圆角）

**卡片：**
- 背景：`#111827`
- 边框：1px `#1F2937`
- Hover：边框变为青蓝色 + 轻微发光

**标签/Badge：**
- 小型标签用于功能标注
- 青蓝色或科技紫，细描边风格

## 六、技术需求

### 6.1 技术选型建议

| 技术 | 选型 | 理由 |
|------|------|------|
| 前端框架 | Next.js 14 (App Router) | SEO友好、性能优、生态好 |
| 样式方案 | Tailwind CSS | 快速实现设计系统，维护方便 |
| 动效库 | Framer Motion | 与React生态集成好，动效能力强 |
| 3D/粒子 | Three.js 或 React Three Fiber | Hero区域3D效果 |
| 表单处理 | React Hook Form | 轻量、高性能 |
| 部署平台 | Vercel 或 阿里云OSS+CDN | 性能稳定，国内访问可选阿里云 |
| 分析工具 | Google Analytics 4 + 百度统计 | 数据双覆盖 |

### 6.2 性能要求

| 指标 | 目标值 |
|------|--------|
| LCP (最大内容绘制) | < 2.5s |
| CLS (累积布局偏移) | < 0.1 |
| FID/INP | < 100ms |
| 移动端 Lighthouse 评分 | > 80 |
| 首屏加载时间（4G） | < 3s |

### 6.3 SEO基础要求

- 每页独立 `<title>` 和 `<meta description>`
- 语义化 HTML 结构（H1/H2/H3 层级正确）
- 图片 `alt` 属性完整
- sitemap.xml 自动生成
- robots.txt 配置
- OG图谱（社媒分享卡片）

### 6.4 响应式断点

| 设备 | 断点 |
|------|------|
| 桌面 | > 1024px |
| 宽屏 | > 1440px |

## 十、风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 产品图/视频素材未就绪 | 阻塞开发 | 先用占位图开发，内容并行准备 |
